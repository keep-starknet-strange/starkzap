## Overview

Starkzap's Payment module enables **cross-chain, multi-token payment acceptance** for your applications. Powered by [Chainrails](https://chainrails.io), it handles the complexity of bridging tokens across different blockchains so you can focus on building your product.

**What users can do:**
- 💳 **Pay from any chain** — Ethereum, Base, Arbitrum, Avalanche, BSC, and more
- 🪙 **Use any token** — USDC, USDT, ETH, STRK, and other popular tokens
- 🌉 **Automatic bridging** — Chainrails finds the best route and handles the cross-chain transfer
- 💰 **Receive on Starknet** — Get settled directly in your preferred token on Starknet

**Real-world analogy:** Think of it like a global payment gateway. Just as Stripe lets users pay with any card and you receive money in your bank, Chainrails lets users pay from any blockchain and you receive tokens on Starknet — all automatically.

<Info>
  The Payment module is **optional** and only activates when you provide payment configuration. Your app works fine without it.
</Info>

## Configuration

### Getting Started

First, get your API key from [Chainrails Dashboard](https://dashboard.chainrails.io):

```typescript
import { StarkZap } from "starkzap";

const sdk = new StarkZap({
  network: "mainnet",
  payment: {
    apiKey: "cr_live_...",      // Your Chainrails API key
    environment: "production",  // Optional: "staging" for testing
  },
});
```

### Configuration Options

| Option | Type | Required | Description |
|--------|------|----------|-------------|
| `apiKey` | `string` | Yes | Your Chainrails API key |
| `environment` | `"production" \| "staging"` | No | API environment (defaults to `"production"`) |

### Accessing the Payment Module

```typescript
const payment = sdk.payment();

// All payment methods are now available
const chains = await payment.getSupportedChains();
```

<Note>
  If `payment` config is not provided, `sdk.payment()` will throw an error, unless when used on the browser. Always check that payment is configured before calling payment methods.
</Note>

## Key Concepts

Before diving into the API, understand these core concepts:

### Chains

Chainrails supports **11+ blockchain networks** across EVM and Starknet:

```typescript
import { PaymentChains } from "starkzap";

// Available chains

PaymentChains.STARKNET_TESTNET

PaymentChains.BASE_TESTNET
PaymentChains.ETHEREUM    // Ethereum Mainnet
PaymentChains.ETHEREUM_TESTNET
PaymentChains.ARBITRUM
PaymentChains.AVALANCHE
PaymentChains.BSC
// ... and more
```

### Tokens

Supported settlement tokens (amount symbols):

```typescript
import { PaymentTokenSymbols } from "starkzap";

PaymentTokenSymbols.USDC  // USD Coin
PaymentTokenSymbols.USDT  // Tether
PaymentTokenSymbols.ETH   // Ethereum
PaymentTokenSymbols.STRK  // Starknet Token
PaymentTokenSymbols.WBTC  // Wrapped Bitcoin
```

### Bridges

Cross-chain bridge protocols supported:

```typescript
import { PaymentBridges } from "starkzap";

PaymentBridges.ACROSS   // Across Protocol
PaymentBridges.CCTP     // Circle's CCTP
PaymentBridges.GATEWAY  // Gateway.fm
PaymentBridges.RHINOFI  // RhinoFi
PaymentBridges.STARKGATE // Starkgate
```

### Payment Intents

A **Payment Intent** represents a concrete payment flow:

1. **PENDING** — Intent created, waiting for user deposit
2. **FUNDED** — Deposit detected on source chain
3. **INITIATED** — Cross-chain bridge transfer started
4. **COMPLETED** — Funds settled on destination chain
5. **EXPIRED** — TTL exceeded without funding

### Sessions

A **Session** binds a recipient + token + amount together. This simplifies the payment flow — payers only need to choose their source chain and token.

## Payment Sessions

Create a payment session to simplify the payer experience. Instead of specifying all payment details, payers just scan a QR code or open a link.

### Creating a Session

```typescript
const session = await payment.createSession({
  recipient: "0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb",
  token: "USDC",
  destinationChain: "STARKNET",
  amount: "100.00",
});

console.log(session.sessionToken);   // "tok_..."
console.log(session.amount);         // "100.00"
```

**Parameters:**
- `recipient` (required) — Wallet address to receive funds on destination chain
- `token` (required) — Token symbol (e.g., "USDC", "ETH")
- `destinationChain` (required) — Chain to receive funds on
- `amount` (required) — Payment amount (string)

### Payment Modal

The **Payment Modal** provides a beautiful, embedded UI for completing cross-chain payments. Users can select their preferred source chain and token.

```typescript
// Create a modal handle
const modalHandle = payment.modal({
  sessionToken: session.sessionToken,
  amount: "100.00",
  platform: "web",  // or "mobile"
});

// Trigger the modal
const success = await modalHandle.pay();

if (success) {
  console.log("Payment completed successfully!");
} else {
  console.log("Payment was cancelled or failed");
}
```

**Modal Flow:**
1. User clicks "Pay" button
2. Modal opens with available payment options
3. User selects source chain and token
4. User completes the cross-chain transfer
5. Modal closes and returns `true` on success, `false` on cancel

<Info>
  The payment modal handles the entire cross-chain swap flow — users pay in their preferred token on any supported chain, and you receive the settlement token on your target chain.
</Info>

## Quotes

Quotes tell you how much a cross-chain payment will cost in terms of fees and final amount.

### Get All Quotes

Get quotes from **all possible source chains** for a destination:

```typescript
const quotes = await payment.getAllQuotes({
  destinationChain: "STARKNET",
  tokenOut: "0x053c91253...",  // USDC on Starknet
  amount: "10",
  recipient: "0xabc...",
});

// Returns quotes from multiple source chains
console.log(quotes[0].sourceChain);      // "BASE_MAINNET"
console.log(quotes[0].totalFee);         // "100000"
console.log(quotes[0].bridge);           // "ACROSS"
console.log(quotes[0].paymentOptions);   // [{ token: "USDC", fee: "..." }]
```

### Get Best Quote

Get the single **best quote** across all bridges:

```typescript
const bestQuote = await payment.getBestQuote({
  sourceChain: "BASE",
  destinationChain: "STARKNET",
  tokenIn: "0xtokenIn...",
  tokenOut: "0xtokenOut...",
  amount: "100",
  recipient: "0xrecipient...",
});

console.log(bestQuote.totalFee);   // Lowest fee
console.log(bestQuote.bridge);     // Best bridge for this route
```

### Get Quotes from All Bridges

Get quotes from **all available bridges** for a specific route:

```typescript
const quotes = await payment.getQuotesFromAllBridges({
  sourceChain: "ETHEREUM",
  destinationChain: "STARKNET",
  tokenIn: "0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48", // USDC
  tokenOut: "0x053c91253...", // USDC on Starknet
  amount: "1000",
  recipient: "0xrecipient...",
  excludeBridges: "CCTP",  // Optional: exclude specific bridges
});
```

### Get Quote from Specific Bridge

Get a quote from a **specific bridge**:

```typescript
const quote = await payment.getQuoteFromBridge({
  sourceChain: "BASE",
  destinationChain: "STARKNET",
  tokenIn: "0xtokenIn...",
  tokenOut: "0xtokenOut...",
  amount: "100",
  bridge: "ACROSS",
  recipient: "0xrecipient...",

```

## Payment Intents

Payment Intents represent the actual payment transaction. Once created, users fund the intent and Chainrails settles it on the destination chain.

### Creating an Intent

```typescript
const intent = await payment.createIntent({
  sender: "0xsender...",
  amount: "10",
  tokenIn: "0xtokenIn...",
  amountSymbol: "USDC",
  source_chain: "BASE",
  destination_chain: "STARKNET",
  recipient: "0xrecipient...",
  refund_address: "0xsender...",
  metadata: {
    description: "Order #42",
    reference: "order-42",
  },
});

console.log(intent.id);              // 1
console.log(intent.intent_address);  // "0xintent..."
console.log(intent.intent_status);   // "PENDING"
```

### Getting an Intent

```typescript
// By ID
const intent = await payment.getIntent("1");

// By on-chain address
const intent = await payment.getIntentByAddress("0xintent...");

// For a sender
const intents = await payment.getIntentsForSender("0xsender...");
```

### Listing Intents

```typescript
const result = await payment.listIntents({
  page: 1,
  limit: 10,
  status: "PENDING",  // Optional: filter by status
});

console.log(result.data);   // Array of intents
console.log(result.total);  // Total count
console.log(result.page);   // Current page
```

### Triggering Processing

Once an intent is funded, trigger processing to settle it:

```typescript
const result = await payment.triggerProcessing(
  "0xintentAddress..."
);

console.log(result.txHash);  // Bridge transaction hash
```

### Intent Status Lifecycle

```typescript
// Update intent status
const updated = await payment.updateIntentStatus(
  "intentId",
  "COMPLETED"
);

// Common status checks
if (intent.intent_status === "PENDING") {
  console.log("Waiting for user to fund...");
} else if (intent.intent_status === "FUNDED") {
  console.log("Deposit received, processing...");
} else if (intent.intent_status === "COMPLETED") {
  console.log("Payment settled!");
}
```

## Complete Example: Checkout Flow

Here's a complete example of implementing a checkout flow:

```typescript
import { StarkZap, PaymentChains, PaymentTokenSymbols } from "starkzap";

// Initialize SDK
const sdk = new StarkZap({
  network: "mainnet",
  payment: {
    apiKey: process.env.CHAINRAILS_API_KEY!,
  },
});

const payment = sdk.payment();

// 1. Create a payment session for a specific amount
async function createPaymentSession(recipient: string, amount: string) {
  const session = await payment.createSession({
    recipient,
    token: PaymentTokenSymbols.USDC,
    destinationChain: PaymentChains.STARKNET,
    amount,
  });
  
  return session;
}

// 2. Create a payment modal
async function openPaymentModal(sessionToken: string, amount: string) {
  const modal = payment.modal({
    sessionToken,
    amount,
    platform: "web",
  });
  
  const success = await modal.pay();
  return success;
}

// 3. Check payment status
async function checkPaymentStatus(intentId: string) {
  const intent = await payment.getIntent(intentId);
  return intent.intent_status;
}

// Usage in your checkout
async function handleCheckout() {
  const recipient = "0xYourWallet...";
  const amount = "25.00";
  
  // Create session
  const session = await createPaymentSession(recipient, amount);
  
  // Open modal
  const success = await openPaymentModal(session.sessionToken, amount);
  
  if (success) {
    console.log("Payment successful!");
    // Check status after a short delay
    setTimeout(async () => {
      const status = await checkPaymentStatus(session.sessionToken);
      console.log("Intent status:", status);
    }, 5000);
  } else {
    console.log("Payment cancelled");
  }
}
```

## Error Handling

The Payment module throws errors when things go wrong:

```typescript
try {
  const session = await payment.createSession({
    recipient: "0xinvalid",
    token: "INVALID",
    destinationChain: "INVALID",
    amount: "100",
  });
} catch (error) {
  if (error.message.includes("API key")) {
    console.log("Check your Chainrails API key");
  } else if (error.message.includes("network")) {
    console.log("Network error, try again");
  } else {
    console.log("Unknown error:", error.message);
  }
}
```
